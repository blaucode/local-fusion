# External Review Disposition — 2026-07-09

An independent agent reviewed the PRD/plan/ADRs. Disposition of each finding, including
what we pushed back on and why. (Traceability doc — findings quoted in summary form.)

| # | Finding | Disposition |
|---|---|---|
| 1 | Parity & ablation lean on an unreliable judge bench; ±0.5 band is inside 0.3–2.0 judge noise | **Accepted, with a stronger fix than proposed.** Port *fidelity* doesn't need judges at all → deterministic record/replay parity (**new ADR-010**, supersedes the judge-score gate; PRD R7 + M3 gates amended). Pushback: a second validated judge does NOT block the migration — it blocks only what genuinely measures quality: the ablation (**ADR-009 amended** with instrument prerequisite: second judge or pre-registered single-judge + test-anchor design; degraded-panel runs invalid by definition) |
| 2 | Account-level provider capacity unarbitrated across clients | **Accepted; severity qualified** (pilot = solo-coder profile, low units; BYOK = per-dev keys). Policy stated in **ADR-002 amendment**: shared key ⇒ shared server (sole arbiter); per-dev servers ⇒ per-dev keys; multiple servers on one key = forbidden configuration |
| 3 | stdio fallback silently loses crash-resilience | **Accepted fully** — sharpest catch. **ADR-002 amendment** names it a partial retreat (timeout fix retained, crash-resilience lost); **ADR-003 amendment** specifies reconnect/idempotency mechanics (derived-stable job keys, `lf_status` rediscovery, conflict semantics) and their HTTP-only validity |
| 4 | No data-governance/egress story; prompt injection unaddressed | **Accepted** — most consequential finding. New **[08-data-governance-and-threat-model.md](./08-data-governance-and-threat-model.md)**: egress per cost profile, retention-terms checklist, new `local-only` profile, threat table. Nuance added: the deterministic test gate is injection-resistant by construction and the gate feeds human PR review (never auto-merge) — now written down as the designed defense instead of implicit |
| 5 | BYOK breaks the "diversity is free" premise; no cost visibility | **Accepted as gap, premise partially pre-answered** (fusion already optional; pilot ships solo). New **PRD R12**: per-run token×price in job results + metrics; BYOK cost line required before pilot widens (PRD pilot-widening preconditions) |
| 2nd tier | Reflexion unvalidated | Accepted in principle — it's P2; when built, same measure-or-flag standard as coder-fusion (noted in 05-roadmap) |
| 2nd tier | Service observability thin | Accepted — structured logs + provider error/latency counters added to **M2 scope** |
| 2nd tier | No `-race`/soak plan for job runner | Accepted — `-race` everywhere + soak test as **M2 exit gate** (testing strategy amended) |
| 2nd tier | Reconnect/idempotency under-specified | Accepted — mechanics drawn in ADR-003 amendment |
| 2nd tier | Pilot sells the gate, not the fusion thesis | Acknowledged as deliberate sequencing, risk accepted openly: the graduation ladder is where the differentiated product lives; a gate-only tool is replicable, the measured pipeline behind the same interface is not |

**Net effect on plan:** no structural change; M0/M1 unchanged. M2 gains the record/replay
harness, observability baseline, and soak-test exit gate. M3's gate is now deterministic.
Two new pilot-widening preconditions (governance sign-off, BYOK cost visibility).
